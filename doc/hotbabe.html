Hotbabe Cluster

Hotbabe consists of 2 primary classes. The HOTBABE_DB class and the HOTBABE class. HOTBABE_DB will read the hotbabe_chess.dat file and build an internal database (as a hash table). This hash table allows us to quickly look up a group name and find text messages and video clips.

Groups and be nested into sub-groups and sub-sub-groups, etc... This structure allows us to traverse a tree of groups to locate a video clip or text message. At each level we choose a random clip (or text message).

HOTBABE is a queue of HOTBABE_EVENT's. Whenever something happens during the game we will insert an event into the hotbabe queue. For example, if the user captures hotbabe's knight. We will insert the event: HOTBABE_PLAYER_CAPTURES_KNIGHT into the queue.

Whenever a video clip finishes playing, we recieve a webcam_event and we dequeue the next event, and play whatever video clip is requested. We also display any text message associated with that video clip.
